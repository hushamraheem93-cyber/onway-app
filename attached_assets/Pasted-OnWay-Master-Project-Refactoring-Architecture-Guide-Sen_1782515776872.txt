قواعد التنفيذ

* لا تنشئ كوداً مكرراً.
* لا تكسر المشروع.
* لا تغيّر قاعدة البيانات إلا إذا لزم الأمر.
* لا تغيّر APIs إلا إذا كان ذلك ضرورياً.
* لا تغيّر Navigation إلا لتحسينه.
* استخدم أفضل ممارسات React Native + Expo + TypeScript + Express + Firebase.

⸻

طريقة التنفيذ

نفّذ العمل مرحلةً مرحلة.

بعد الانتهاء من كل مرحلة:

* تأكد أن المشروع ما زال يعمل.
* أصلح أي خطأ.
* ثم انتقل للمرحلة التالية.

⸻

الهدف النهائي

اجعل المشروع بمستوى تطبيق عالمي جاهز للإنتاج من حيث:

* Architecture
* Code Quality
* Performance
* Security
* Scalability
* UX/UI
* Maintainability
