ONWAY — Design System Refactor (No UI Changes)

Goal

Refactor the entire application’s color system into a single source of truth without changing the current UI or breaking any functionality.

Requirements

1. Single Color Source

Use only:

client/constants/theme.ts

as the only place that defines colors.

No component or screen may define hardcoded colors anymore.

⸻

2. Remove Duplicate Colors

Delete duplicated variables such as:

wayYellow
primaryColor duplicates
local ORANGE constants
local SUCCESS constants
local ERROR constants

Everything must reference:

AppColors
Colors
ORDER_STATUS_COLORS

only.

⸻

#4CAF50

14. Validation

After finishing:

• Search entire project for remaining hardcoded HEX values.

• Search for duplicated color constants.

• Search for duplicate status color maps.

• Search for duplicate button colors.

Everything must come from theme.ts.

⸻

15. Important

Do NOT redesign the UI.

Do NOT change spacing.

Do NOT change typography.

Do NOT change layouts.

Do NOT change animations.

Only refactor and centralize colors.

The application should look exactly the same while becoming fully maintainable.
