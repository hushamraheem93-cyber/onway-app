⚠️ توصيات للتحسين قبل أو بعد الإطلاق

  • Rules file: التطبيق يستخدم Firebase Admin SDK على الخادم فقط. تأكد في Firebase Console أن Firestore Rules تمنع الوصول العام المباشر من العميل.
  • Client SDK usage: راجع Firestore Rules في Firebase Console وتأكد من تقييد الوصول المناسب للقراءة المباشرة
  • JWT_SECRET configured: أضف JWT_SECRET كمتغير بيئة بقيمة عشوائية قوية
  • Firebase client config: تأكد من Firestore Security Rules في Firebase Console لتقييد الوصول المباشر
  • Server console.log count: استبدل console.log غير الضرورية بـ logger مناسب في الإنتاج
  • Request logging: في الإنتاج، قيّد التسجيل للأخطاء فقط أو استخدم نظام logging منفصل
  • Demo stores in database: احذف البيانات التجريبية من قاعدة البيانات قبل الإطلاق
  • Production CORS restriction: في بيئة الإنتاج، أضف ALLOWED_ORIGINS=https://your-domain.replit.app إلى متغيرات البيئة
  • Firestore automatic backups: فعّل Firebase Firestore Scheduled Backups من Firebase Console → Firestore → Backups
